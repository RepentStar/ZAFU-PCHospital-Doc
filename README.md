## 编写规范

Fork 本仓库，将自己的仓库 clone 之本地进行修改，push 到自己的仓库中，再在 github 上创建 PR

~~不要学我直接提交到组织仓库，只是这个仓库平时就我写所以这样方便点~~

commit 前缀规范：
| type | 含义 | 本项目适用场景 |
| -------- | ----------- | ------------------------------ |
| revise | 文档内容变更 | 新增/修改章节 |
| add | 新增文件 | 新增章节文件 |
| chore | 构建/配置变更 | book.toml、GitHub Actions 配置 |
| refactor | 结构调整（不增删内容） | 移动文件、重命名目录 |
